Deep learning workflow steps:¶

Step 1: Define the problem¶

I am using the MNIST dataset, which contains a set of images of handwritten digits.

I am trying to solve the problem of correctly categorizing the digit in each image into their correct category. So this will be a single-label, multiclass classification problem.

The input samples will be the images, and the output will be a set of probabilities, determining which number the image most likely contains.

This dataset contains 70 000 samples (60 000 training and 10 000 test), so data isn't a limiting factor.

Step 2: Choosing measure of success¶

Since the dataset isn't perfectly balanced, I'll be using both precision and recall as a measure of success.

Step 3: Choosing evaluation protocol¶

As a evaluation protocol, I will test the success of k-fold validation, but will maintain a hold-out validation set.

Step 4: Preparing the data¶

Currently, the training and testing labels are stored in a 60 000 and 1 000 element vector ((60 000,) and (1000,) respectively).

The traing and testing images are stored in an array of (60 000, 28, 28) and (1 000, 28, 28). This will be problematic and will need to be changed.

The data has to be reshaped to a form that is acceptable to the network. The network will expect samples as vectors (1D arrays) of floating point values. The pixel map has to be straightened into a line of values.

I will flatten the input array to an array with 783 elements. I will then cast them as floats and rescale to [0,1].

Step 5: Developing a model that does better than a baseline¶

Here I wish to achieve statistical power by developing a small model that is capable of performing better than a random classifier.

This is a multiclass classification problem with 10 distinct classes, so I assume the baseline performance will be an accuracy of 0.1, since there is a 10% likelihood of each sample being classified correctly with a random classifier.

To beat this, I will create a small model that outperforms the 0.1% accuracy.

I will use categorical crossentropy as the loss function as it is most suitable for single-label multi-class classification tasks.

import matplotlib.pyplot as plt

def plot_epochs(x, 
               y_A, style_A, label_A,
               y_B, style_B, label_B,
               title, x_label, y_label):
    plt.clf()
    plt.plot(x, y_A, style_A, label = label_A)
    plt.plot(x, y_B, style_B, label = label_B)
    plt.title(title)
    plt.xlabel(x_label)
    plt.ylabel(y_label)
    plt.legend()
    plt.show()

import numpy as np

# K-Fold validation
# Number of folds
K=5
# Index for validation data
num_val_samples = len(train_images) // K
# Use lowest-loss epoch
num_epochs = lowest_loss
all_acc = []
all_loss = []

for i in range(K):
    print('processing fold: ', i)
    # create set of validation image data
    val_data = train_images[i * num_val_samples: (i+1) * num_val_samples]
    # create set of validation label data
    val_targets = train_labels[i * num_val_samples: (i+1) * num_val_samples]
    # concatenate the remaining partitions
    partial_train_data = np.concatenate([train_images[:i * num_val_samples], 
                                         train_images[(i + 1) * num_val_samples:]],
                                       axis=0)
    partial_train_targets = np.concatenate([train_labels[:i * num_val_samples], 
                                         train_labels[(i + 1) * num_val_samples:]],
                                       axis=0)

    # train and evaluate a fresh model
    k_network = models.Sequential()
    k_network.add(layers.Dense(300, 
                               kernel_regularizer = regularizers.L2(0.0000001),
                               activation='relu', input_shape=(28*28,)))
    k_network.add(layers.Dropout(0.25))
    k_network.add(layers.Dense(300, 
                               kernel_regularizer = regularizers.L2(0.0000001),
                               activation='relu'))
    k_network.add(layers.Dropout(0.25))
    k_network.add(layers.Dense(300, 
                               kernel_regularizer = regularizers.L2(0.0000001),
                               activation='relu'))
    k_network.add(layers.Dropout(0.25))
    k_network.add(layers.Dense(10, activation='softmax'))
    
    k_network.compile(optimizer='rmsprop',
                    loss='categorical_crossentropy',
                    metrics=['accuracy'])
    # Train model
    k_network.fit(partial_train_data, 
                  partial_train_targets, 
                  epochs=num_epochs, 
                  batch_size=256, 
                  verbose=0)

    # Evaluate model
    test_loss, test_acc = k_network.evaluate(val_data, val_targets, verbose=0)
    all_acc.append(test_acc)
    all_loss.append(test_loss)

print('Fold accuracies: ', all_acc)
print('Mean accuracy: ', np.mean(all_acc))
print('Fold losses: ', all_loss)
print('Mean loss: ', np.mean(all_loss))

Conclusion¶

In this project, I used the MNIST digit dataset to train a model for classifying images of digits.

The steps I took were as follows:

  1. Created a small model to beat the baseline
  2. Increased the model size to overfit
  3. Slightly reduced model size, while still overfitting
  4. Use a validation set to compare performance of loss and accuracy
  5. Added kernel regularization and dropout layers to reduce overfitting
  6. Finalized with K-fold validation to obtain an average of the losses and accuracies

The K-fold validation returned an average accuracy of 97.8% and a loss of 9.8%.

Although these is still overfitting in the model, since the training accuracy is higher than the testing accuracy, the techniques I've implemented to improve the models performance (dropout and regularization) improved the reliability of the model.
